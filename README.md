# Project NEO AI

Neo is a next-generation AI assistant built for the web and the future BLIX ecosystem. It is designed around voice interaction, intelligent automation, memory, and a modern user experience so it can become a powerful personal AI companion.

## Vision

Project NEO AI is the starting point for Neo, the assistant layer of BLIX. The long-term goal is to create an AI companion that can understand user intent, help control apps and workflows, remember useful context with permission, and feel fast, polished, and dependable.

BLIX is the broader ecosystem that Neo will eventually support. Neo should be able to act as the friendly interface between the user and BLIX features, including productivity tools, automation, system actions, and personalized assistance.

## Core Goals

- Build a clean, scalable foundation for Neo.
- Keep the user experience simple, modern, and responsive.
- Support text-first interaction before expanding into voice and automation.
- Design memory and personalization with privacy and user control in mind.
- Keep project modules small, readable, and easy to improve.

## Planned Modules

- `core`: intent handling, assistant logic, and command routing.
- `ui`: the web interface for chat, controls, and BLIX surfaces.
- `memory`: user-approved context, preferences, and recall.
- `voice`: speech input, speech output, and voice state handling.
- `automation`: safe actions, workflows, and integrations.
- `docs`: architecture notes, product decisions, and setup guides.

## Development Principles

- Make small, logical changes with meaningful commit messages.
- Preserve working functionality unless a change is required.
- Keep code modular, documented where helpful, and production-minded.
- Avoid committing secrets, API keys, local credentials, or generated clutter.
- Prefer clear architecture decisions over rushed feature additions.

## Roadmap

1. Define the initial product direction and repository structure.
2. Add a minimal runnable application shell.
3. Implement the first text-based Neo interaction loop.
4. Add a polished BLIX-inspired interface.
5. Introduce safe memory and settings.
6. Expand toward voice, automation, and deeper BLIX integration.

## Status

This repository is in the foundation stage. The first priority is to establish a stable project structure and document the direction before adding major features.
