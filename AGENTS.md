# Project NEO AI - AGENTS.md

## Project Vision

Project NEO AI is a next-generation AI assistant that will eventually become the AI core of the BLIX ecosystem.

The primary goal is to build a premium, futuristic, voice-first AI assistant with a beautiful user interface, modular architecture, and scalable codebase.

---

# Development Principles

- Quality over speed.
- Keep the code modular.
- Never break existing functionality.
- Reuse components whenever possible.
- Maintain clean architecture.
- Keep performance high.
- Write production-quality code.

---

# UI Design

The UI should feel like a futuristic operating system.

Use:

- Glassmorphism
- Blue/Cyan/Green glow
- Dark mode
- Smooth animations
- Rounded corners
- Soft shadows
- Minimalistic layout

Avoid:

- Bright colors
- Cluttered layouts
- Heavy libraries

---

# Orb

The orb is the center of Neo.

States:

- idle
- listening
- thinking
- speaking
- error

Animations should be smooth and GPU-friendly.

---

# Code Style

- Modular
- Reusable
- Well structured
- Easy to maintain
- No duplicate code

---

# Git

Create small logical commits.

Examples:

feat(ui): create Neo orb

feat(chat): improve interface

style(ui): improve glass effects

refactor(core): simplify architecture

docs(readme): improve documentation

Never combine unrelated changes into one commit.

---

# Before Coding

Always:

1. Analyze the repository.
2. Understand the architecture.
3. Plan the change.
4. Implement.
5. Test.
6. Commit.

---

# Never

- Delete working code without reason.
- Add unnecessary dependencies.
- Break existing features.
- Leak secrets or API keys.
- Change architecture without justification.

---

# Long-term Goal

Build Neo into a premium AI assistant that powers the BLIX ecosystem while maintaining clean, scalable, production-ready software.
